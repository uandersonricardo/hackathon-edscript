    # Retrieve Odds - Groups¶

    Bet Type Groups¶

    describe market categories in sports betting. They are metadata containers for odds, grouping similar bet types (e.g., “Full Time Result”, “Total Goals”, “Handicap”).

    They tell you: - Which type of bet is available, - How it should be displayed (ordering, compact view), - Linkage to its main bet type group.

    Key Fields (from BtgSummarySerializer)¶

    • betTypeGroupId → Unique identifier of the bet type group.
    • betTypeGroupName → Localized name of the bet type group.
    • orderBy / asianOrderBy → Ordering indices (regular and Asian-specific).
    • promoted → Whether this group is promoted/highlighted.
    • mainBtgId → Reference to parent/main group.
    • mainBtgName → Name of the parent group.
    • mainBtgOrder → Ordering index of the parent group.
    • hsd (HeaderSetDetailDto[]) → Array of header-set details (e.g., for displaying structured betting lines).
    • cv / cvp / cvOrd → Compact view fields.
    • dt / ddt → Display type identifiers.

    Example (simplified)¶

    {
  "betTypeGroupId": 12,
  "betTypeGroupName": "Full Time Result",
  "orderBy": 1,
  "promoted": true,
  "mainBtgId": 1,
  "mainBtgName": "Main Markets",
  "mainBtgOrder": 0,
  "hsd": [
    { "id": 101, "name": "1X2", "viewIndex": 1 }
  ],
  "cv": false,
  "cvp": null,
  "cvOrd": 1,
  "dt": 100,
  "ddt": null
}

    Fixture Bet Type Groups¶

    Purpose¶

    FixtureBtg represents a specific instance of a BetTypeGroup attached to a fixture (match/event).
    While BetTypeGroups are general market definitions, Fixture Bet Type Groups actually hold the odds (fos) offered in that match.

    Key Fields (inferred from serializers & models)¶

    • btgId / btgName → Reference to the BetTypeGroup.
    • fos[] (FixtureOdd[]) → The actual odds for this fixture in this group.
    • order → Market ordering index inside the fixture.
    • displayTypeId → How the market is displayed in frontend.
    • compactView fields inherited from BtgSummary.
    • Possibly enriched by MatchData (e.g., score/period info for context).

    Example (simplified)¶

    {
  "btgId": 12,
  "btgName": "Full Time Result",
  "order": 1,
  "fos": [
    {
      "oddId": 501,
      "selectionName": "Home",
      "oddValue": 1.85
    },
    {
      "oddId": 502,
      "selectionName": "Draw",
      "oddValue": 3.20
    },
    {
      "oddId": 503,
      "selectionName": "Away",
      "oddValue": 4.50
    }
  ]
}

    Relationship Between Them¶

    • BetTypeGroups (BtgSummary) = Global market metadata (labels, order, display settings).
    • FixtureBetTypeGroups (FixtureBtg) = Fixture-specific markets that reference a BetTypeGroup and contain actual odds (fos).

    Think of it as:
    📚 BetTypeGroups = dictionary/blueprints of markets.
    📖 FixtureBetTypeGroups = actual market entries in a match, with odds filled in.
